# Python 코드 설명기: 데모 가이드

이 문서는 'Python 코드 설명기' 프로젝트의 개요, 사용법, 그리고 개발 과정을 담고 있습니다.

---

## 1. 프로젝트 소개

#### 개요

이 프로젝트는 Google의 Gemini AI 모델을 활용하여, 사용자가 입력하는 Python 코드의 기능과 목적을 한국어로 간결하게 설명해주는 Command Line Interface(CLI) 애플리케이션입니다. 복잡한 코드의 핵심을 빠르게 파악하거나, 다른 사람이 작성한 코드를 이해하는 데 도움을 주는 것을 목표로 합니다.

#### AI 모델 선정 이유

사용한 모델은 **`gemini-2.5-flash`** 입니다. 이 모델을 선택한 이유는 다음과 같습니다.

1.  **속도와 성능의 균형:** 실시간으로 사용자 입력을 처리하고 빠르게 결과를 보여주는 CLI 환경에 적합하도록, 응답 속도가 빠르면서도 우수한 성능을 내는 'flash' 모델이 가장 적합하다고 판단했습니다.
2.  **최신 모델 활용:** Google에서 제공하는 최신 모델 중 하나로, 복잡한 코드의 맥락까지 잘 이해하고 자연스러운 한국어 설명을 생성하는 능력이 뛰어납니다.

## 2. 라이브 데모 시연 가이드

애플리케이션의 핵심 기능은 터미널에서 직접 실행하며 확인할 수 있습니다.

#### 실행 절차

1.  아래 명령어를 터미널에 입력하여 애플리케이션을 실행합니다.

    ```bash
    python app.py
    ```

2.  `설명할 Python 코드를 입력하세요:` 라는 안내가 나오면, 분석하고 싶은 코드 한 줄을 입력하고 Enter를 누릅니다.

#### 시연 예시

> **입력 예시:**
>
> ```python
> print([i for i in range(10) if i % 2 == 0])
> ```

> **출력 결과:**
>
> ```text
> --- Gemini의 설명 ---
> 0부터 9까지의 숫자 중 짝수만 포함하는 리스트를 생성하고 출력하는 코드입니다.
> ---------------------
> ```

## 3. 핵심 코드 리뷰 및 개발 회고

#### 핵심 로직: API 호출부

이 애플리케이션의 핵심은 사용자의 입력을 받아 API에 요청을 보내는 부분입니다. 아래는 `app.py`의 해당 코드입니다.

```python
# ... (전략) ...

# 3. 프롬프트 구성 
# "Python 코드 설명기" 기능 구현 
prompt = f"""
당신은 전문 Python 개발자입니다. 
다음 Python 코드의 역할을 한글로 간결하게 설명해주세요.

[코드]
{user_input}

[설명]
"""

# 4. API 호출
response = model.generate_content(prompt)

# 5. 결과 출력
print("--- Gemini의 설명 ---")
print(response.text)

# ... (후략) ...
```

**작동 방식:**

1.  `prompt` 변수에 AI에게 역할을 부여하고, 사용자가 입력한 코드(`user_input`)를 포함시켜 전체 요청 메시지를 만듭니다.
2.  `model.generate_content(prompt)`를 통해 이 프롬프트를 Gemini API로 전송합니다.
3.  API가 생성한 응답(설명 텍스트)은 `response.text`에 담겨 있으며, 이를 화면에 출력하여 사용자에게 보여줍니다.

#### 개발 회고: 기술적 어려움과 해결 과정

> **질문: 이 데모를 만들면서 기술적으로 가장 어려웠던 점은 무엇이었고, 그 문제를 어떻게 해결했나요?**

**답변:**
가장 어려웠던 점은 **처음 Google AI API를 연동할 때 발생했던 모델 관련 오류**였습니다. `app.py`를 만들고 API를 호출하는 코드를 작성했는데, 자꾸 모델을 찾을 수 없다는 에러가 발생했습니다. API 키가 잘못되었는지, 아니면 모델 이름을 잘못 입력했는지 원인이 불분명해서 막막했습니다.

이 문제를 해결하기 위해, 저는 **문제를 작은 단위로 나누어 접근하는 방법을 선택**했습니다. 즉, `app.py`의 복잡한 코드에서 바로 문제를 해결하려 하지 않고, **`check_model.py`라는 아주 작은 테스트용 스크립트를 별도로 작성**했습니다. 이 파일의 목적은 단 하나, '내 API 키로 접근 가능한 모델 목록을 전부 출력해보자'였습니다. 

이 스크립트를 실행해보니 제가 사용하려던 모델의 정확한 이름이 `gemini-2.5-flash`가 아니라 `models/gemini-2.5-flash`라는 것을 알 수 있었습니다. 이 작은 검증 과정을 통해 문제의 원인을 명확히 찾아냈고, `app.py`에 올바른 모델 이름을 적용하여 문제를 해결할 수 있었습니다. 이 경험을 통해 **복잡한 문제에 직면했을 때, 기능을 최소 단위로 분리하여 테스트하는 방식이 얼마나 효과적인지** 깨닫게 되었습니다.
